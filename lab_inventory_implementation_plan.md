## 1. システム概要

ラボで使用する物品（試薬・消耗品など、〜30品目）の在庫状況をリアルタイムで把握・管理するWebアプリケーション。在庫管理係が日常的に運用することを前提とし、在庫入力・発注管理・一覧表示の3機能を中心に構築する。

### 基本方針

- **データ保存：** ブラウザのLocalStorage（サーバー不要、即日利用可能）
- **実装形態：** 単一HTMLファイル（Coworksで管理しやすい構成）
- **スケール感：** 30品目以下のため、LocalStorageの容量（〜5MB）で十分に対応可能
- **将来拡張：** 複数ユーザー対応・CSV一括管理・カテゴリ分類を想定した設計とする（第9章参照）

## 2. 機能要件

### 2-1. 在庫一覧画面（メイン画面）

在庫状況をひと目で把握できるダッシュボード。

**表示項目（テーブル形式）：**

| 列 | 内容 |
|---|---|
| 物品名 | 試薬・消耗品などの名称 |
| カテゴリ | 試薬 / 消耗品 / 器具 / その他 |
| 現在庫数 | 数値（単位付き） |
| 単位 | 本 / 箱 / 袋 / 個 など |
| 発注点（閾値） | この数以下になったら警告 |
| 発注状況 | 未発注 / 発注済み |
| 最終更新日 | 在庫数を最後に変更した日時 |
| 操作 | 編集・削除ボタン |

**カラーコーディング（在庫状況の視覚的表示）：**

- 🔴 **赤（警告）：** 現在庫数 ≤ 発注点　かつ　発注未済
- 🟡 **黄（注意）：** 現在庫数 ≤ 発注点　かつ　発注済み
- 🟢 **緑（正常）：** 現在庫数 > 発注点

**フィルタ・ソート機能：**
- カテゴリでフィルタリング
- 「要注意のみ表示」トグル（赤・黄のみ表示）
- 物品名・在庫数・更新日でソート可能

**サマリーバナー（画面上部）：**
- 全物品数 / 要確認数（赤） / 発注済み数（黄）をカード形式で表示

---

### 2-2. 在庫入力フォーム

新規物品の登録・既存物品の在庫数更新に使用する。

**新規登録フィールド：**

| フィールド | 型 | 必須 | 備考 |
|---|---|---|---|
| 物品名 | テキスト | ✅ | |
| カテゴリ | セレクト | ✅ | 試薬 / 消耗品 / 器具 / その他 |
| 現在庫数 | 数値 | ✅ | 0以上 |
| 単位 | テキスト | ✅ | 例：本、箱、袋 |
| 発注点（閾値） | 数値 | ✅ | この値以下で警告 |
| 発注先業者 | テキスト | ❌ | 任意 |
| 備考 | テキスト（複数行） | ❌ | |

**在庫数の更新方式（2通り）：**
1. **直接入力：** 現在庫数を数値で上書き入力
2. **増減入力：** 「+入荷」「-使用」ボタンで差分を入力 → 自動計算

---

### 2-3. 発注管理画面

要発注品の確認と発注ステータスの管理。

**表示内容：**
- 発注点を下回った物品の一覧（カテゴリ・業者でグループ化表示）
- 各物品に「発注済みにする」ボタン → ステータス変更 + 発注日を自動記録
- 「発注済み」物品に「納品済み（在庫に戻す）」ボタン → 在庫数更新フォームへ遷移

**発注履歴：**
- 発注日・物品名・数量（任意入力）・業者名の記録
- 過去10件の発注履歴を表示

---

### 2-4. データ管理機能

| 機能 | 実装時期 | 説明 |
|---|---|---|
| CSVエクスポート | **v1（初期実装）** | 現在の在庫一覧をCSVファイルでダウンロード |
| CSVインポート | **v1（初期実装）** | CSVファイルから物品を一括登録・更新 |
| データバックアップ | v1 | LocalStorageの内容をJSONとしてエクスポート |
| データリストア | v1 | JSONファイルからデータを復元 |
| 複数ユーザー対応 | **v2（将来）** | バックエンド移行時に実装 |

#### CSVインポートの列仕様（v1）

インポート用CSVは以下の列構成で受け付ける。Excelで編集して取り込む運用を想定。

```
物品名,カテゴリ,現在庫数,単位,発注点,発注先業者,備考
エタノール 500mL,試薬,8,本,3,和光純薬,冷暗所保管
ピペットチップ 1000μL,消耗品,50,箱,10,ニッポンジーン,
```

- **新規登録：** 物品名が存在しない → 新規追加
- **更新：** 物品名が既存と一致 → 在庫数・発注点などを上書き（確認ダイアログあり）
- **バリデーション：** 必須列（物品名・在庫数・単位・発注点）が空の行はスキップ＋エラー表示

## 4. データ構造設計

### 物品マスタ（items）

```json
{
  "id": "uuid",
  "name": "エタノール 500mL",
  "category": "試薬",
  "stock": 8,
  "unit": "本",
  "threshold": 3,
  "supplier": "和光純薬",
  "notes": "冷暗所保管",
  "orderStatus": "none",
  "orderDate": null,
  "createdAt": "2026-03-01T00:00:00Z",
  "updatedAt": "2026-03-01T00:00:00Z"
}
```

**orderStatus の値：**
- `"none"` → 未発注（在庫十分 or 要発注だが未着手）
- `"ordered"` → 発注済み（納品待ち）

### 発注履歴（orderHistory）

```json
{
  "id": "uuid",
  "itemId": "uuid",
  "itemName": "エタノール 500mL",
  "orderedAt": "2026-03-01T00:00:00Z",
  "quantity": 5,
  "unit": "本",
  "supplier": "和光純薬",
  "note": ""
}
```

## 6. UIデザイン方針

### カラーパレット

| 用途 | 色 | 説明 |
|---|---|---|
| プライマリ | `#2563EB`（青） | ボタン・ヘッダー |
| 警告（赤） | `#FEE2E2` / `#DC2626` | 在庫不足・未発注 |
| 注意（黄） | `#FEF9C3` / `#CA8A04` | 在庫不足・発注済み |
| 正常（緑） | `#DCFCE7` / `#16A34A` | 在庫十分 |
| 背景 | `#F8FAFC` | ページ背景 |

### UX原則

- テーブル行全体をクリックで在庫数クイック編集（インライン編集）
- 発注点を下回ったアイテムは行全体が色付き → 一覧でひと目でわかる
- モバイル対応（レスポンシブ）: 管理者がスマホで確認できる

## 7. Coworksへの実装指示（プロンプト例）

Coworksで実装を依頼する際は、以下のプロンプトを使用すること。

---

### 【プロンプト本文】

```
以下の仕様に従い、ラボの物品在庫管理システムを単一のHTMLファイルとして実装してください。

## 技術要件
- 単一HTMLファイル（HTML + CSS + JavaScript をすべて1ファイルに収める）
- Tailwind CSS を CDN から読み込む（https://cdn.tailwindcss.com）
- Lucide Icons を CDN から読み込む
- データはすべて LocalStorage に保存（キー名: "lab_inventory_items", "lab_inventory_orders"）
- ライブラリ追加禁止（Vanilla JSのみ）

## 画面構成（SPA: ページ遷移はJS制御）
1. 在庫一覧画面（デフォルト）
2. 発注管理画面
3. 物品登録・編集フォーム

## 在庫一覧画面の仕様
- 画面上部にサマリーカード3枚（全物品数 / 要確認数[赤] / 発注済み数[黄]）
- カテゴリフィルタ（全て/試薬/消耗品/器具/その他）
- 「要注意のみ表示」トグルボタン
- 在庫テーブル列：物品名・カテゴリ・在庫数・単位・発注点・発注状況・最終更新・操作
- 行カラーコーディング：
  - 在庫≦発注点 かつ 未発注 → 行背景 #FEE2E2（赤）
  - 在庫≦発注点 かつ 発注済み → 行背景 #FEF9C3（黄）
  - 在庫 > 発注点 → 通常（白）
- 操作列：「在庫更新」ボタン（インラインで在庫数を±入力できるモーダル）、「編集」「削除」ボタン
- 「＋物品を追加」ボタン → 物品登録フォームへ

## 在庫更新モーダルの仕様
- モーダルタイトル：「在庫数を更新（物品名）」
- 現在の在庫数を表示
- 入力方式①：「現在庫数を上書き」→ 数値入力欄
- 入力方式②：「入荷」ボタン → 数値を加算、「使用」ボタン → 数値を減算
- 更新後に updatedAt を現在日時で自動更新

## 発注管理画面の仕様
- 発注点を下回った物品（orderStatus: "none"）を一覧表示
- 各行に「発注済みにする」ボタン → orderStatus を "ordered"、orderDate を現在日時に変更
- 「発注済み」物品（orderStatus: "ordered"）の一覧を別セクションで表示
- 各発注済み行に「納品完了・在庫更新」ボタン → 在庫更新モーダルを開き、更新後に orderStatus を "none" にリセット

## 物品登録・編集フォームの仕様
- フィールド：物品名（必須）・カテゴリ（セレクト）・現在庫数（必須・数値）・単位（必須）・発注点（必須・数値）・発注先業者（任意）・備考（任意）
- 新規・編集両対応（編集時は既存データを初期値としてフォームに表示）
- バリデーション：必須項目が空の場合はエラーメッセージ表示

## データ構造（LocalStorage）
items 配列:
{
  id, name, category, stock, unit, threshold,
  supplier, notes, orderStatus("none"|"ordered"),
  orderDate, createdAt, updatedAt
}

## サンプルデータ
初回アクセス時に以下のサンプルデータを自動挿入すること:
- エタノール 500mL（試薬・在庫2・閾値3）→ 赤表示になるよう設定
- ピペットチップ 1000μL（消耗品・在庫15・閾値10）→ 黄表示（発注済み）
- マイクロチューブ 1.5mL（消耗品・在庫50・閾値20）→ 緑表示
- PBS バッファ 500mL（試薬・在庫5・閾値5）→ 赤表示になるよう設定

## UIデザイン要件
- ヘッダー：ダークブルー背景（#1E3A5F）、白文字、アプリ名と🧪アイコン
- ナビゲーション：タブUI（在庫一覧・発注管理・物品登録）
- フォントはシステムフォント（sans-serif）
- ボタンはホバー時に色を変化させる
- テーブルはゼブラストライプ（偶数行に薄グレー）
- モバイル対応：画面幅が小さい場合はテーブルを横スクロール対応

## その他
- 削除時は確認ダイアログ（confirm()）を表示
- CSVエクスポートボタン（在庫一覧画面の右上）→ 現在表示中のデータをCSVダウンロード
- CSVインポートボタン（在庫一覧画面）→ ファイル選択ダイアログを開き、以下の列構成のCSVを読み込む
  - 列構成: 物品名,カテゴリ,現在庫数,単位,発注点,発注先業者,備考
  - 物品名が既存と一致する場合は上書き確認、一致しない場合は新規追加
  - 必須列（物品名・現在庫数・単位・発注点）が空の行はスキップしエラー行数を表示
- データバックアップボタン（在庫一覧画面）→ LocalStorageの全データをJSONとしてダウンロード
```

## 9. 将来の拡張計画（v2以降）

v1はLocalStorageベースの単一ユーザー運用を前提とする。以下はユーザーから明示的に要望があった拡張項目と、その実装方針。

### v1.5：CSVインポート（初期実装に含む）

CSVインポートは30品目規模でも初期登録や棚卸時に有効なため、v1に含めて実装する（第2-4章参照）。

### v2：複数ユーザー対応（最優先の将来課題）

現状のLocalStorageはブラウザ単位でのデータ保存のため、複数の端末・ユーザーでデータを共有できない。複数人運用には以下の移行が必要。

| ステップ | 内容 |
|---|---|
| バックエンド選定 | **Supabase（推奨）**：無料プランあり・SQLベース・RLS（行レベルセキュリティ）対応 |
| データ移行 | v1のLocalStorageデータをJSONエクスポート → Supabaseにインポート |
| 認証 | Supabase Authでメール認証またはGoogle認証を追加 |
| リアルタイム同期 | Supabase Realtimeを使用し、複数ユーザーの更新を即時反映 |

> **⚠️ Coworksでの実装時の注意：** v2への移行はHTML単一ファイルの構成を変更する必要があるため、v2実装時に改めて計画書を作成してCoworksに依頼すること。

### その他の拡張候補

| 優先度 | 拡張項目 | 実装方針 |
|---|---|---|
| 中 | カテゴリのカスタマイズ | カテゴリをデータとして管理し、追加・削除を可能にする |
| 中 | 使用期限・ロット番号管理 | データ構造に expiryDate, lotNumber フィールドを追加。期限が近い物品を黄色で警告 |
| 低 | 在庫変動グラフ | 更新ログを蓄積し、Chart.js で時系列グラフを表示 |
| 低 | メール・Slack通知 | 在庫不足時に自動通知（バックエンド必須） |
